# SkillFest Developer Event: Platform Guide

Welcome to **SkillFest 2025**, a week-long open source contribution program designed to showcase development skills and identify top talent for our team. This document explains how the platform works and how the event is structured.

---

## 🚀 Event Overview

SkillFest is divided into two primary tracks, catering to different experience levels:

1.  **Developer Challenge**: An open-source contribution phase for experienced developers.
2.  **Fresher Developer Program**: An application-based track for those starting their career.

---

## 🛠 Hand-on: Developer Challenge

The Developer Challenge is the heart of SkillFest. It focuses on real-world contributions to the **nst-sdc** organization on GitHub.

### 1. Registration & Participation
- Participants sign in using their **GitHub account**.
- Once signed in, they gain access to a curated list of open issues across various repositories in the `nst-sdc` organization.

### 2. Contribution Phase
- **Find an Issue**: Browse the list of available challenges (bugs, enhancements, features) fetched directly from GitHub.
- **Submit PRs**: Fork the repository, work on the issue, and submit a Pull Request (PR) to the original repository.
- **Review**: Our team reviews the PRs for code quality, complexity, and correctness.

### 3. Scoring & Leaderboard
Every contribution earns points. The system automatically tracks merged PRs and updates your standing.
- **Points**: Awarded based on the difficulty and quality of the contribution.
- **Ranks**: Participants are categorized into levels: `Newcomer` → `Beginner` → `Intermediate` → `Advanced` → `Expert`.
- **Visibility**: The leaderboard shows real-time standings of all contributors.

### 4. Selection Criteria
The **Top 15** performers on the leaderboard are selected for potential positions. Criteria include:
- Minimum of **3 quality Pull Requests**.
- Evaluation of code complexity and impact.
- Adherence to best practices.

---

## 🎓 Fresher Developer Program

For those with less experience, SkillFest offers a structured application process.

### 1. Application Process
- Applicants fill out a detailed form including GitHub profile, experience, portfolio, and "Why join?" statement.
- Applications are recorded securely in our database.

### 2. Admin Review
- Managers use the **Admin Dashboard** to review candidates.
- Each application can be marked as `Pending`, `Approved`, or `Rejected`.
- Admins can view full resumes, portfolios, and contact details directly from the dashboard.

---

## ⚙️ Technical Architecture

- **Frontend**: Built with Next.js (React), Tailwind CSS, and Lucide icons for a premium UI/UX.
- **Backend/DB**: Powered by **Firebase** for real-time application tracking and leaderboard data.
- **Integration**: Deeply integrated with the **GitHub API** to fetch issues and track pull request status.
